# 請求受取太郎 ― サービスの強み

## サービス概要

**請求受取太郎**は、取引先からの請求書を専用URLで簡単に受け取り・管理できるクラウド型請求書受領サービスです。  
「受け取る側」の手間を最小化することに特化した設計で、経理業務のDXを推進します。

---

## 強み一覧

### 1. 取引先にアカウント不要 ― 専用リンクだけで完結

- 取引先ごとにユニークなアップロード専用URLを自動発行
- 取引先はアカウント登録やログインが一切不要
- URLを共有するだけで、すぐに請求書のアップロードが可能
- **導入障壁を極限まで下げ、取引先への負担ゼロ**を実現

### 2. シンプルで直感的なUI/UX

- 取引先向けアップロード画面はファイル選択・金額・日付の3ステップのみ
- ドラッグ&ドロップによるファイルアップロードに対応
- PDF・PNG・JPG形式をサポートし、多様な請求書フォーマットに対応
- モバイルレスポンシブ対応で、スマートフォンからもアップロード可能

### 3. リアルタイムな請求書管理ダッシュボード

- 受け取った請求書を一覧で即座に確認
- **月別フィルタリング**で期間を絞り込み
- **ステータス別フィルタリング**（未振込 / 振込済）で支払い状況を可視化
- 合計金額・未払い金額をリアルタイムに自動集計
- ワンクリックでのステータス切り替えにより、支払い漏れを防止

### 4. マルチテナント対応 ― 組織単位の安全なデータ管理

- 組織（Organization）単位でのデータ分離を標準装備
- Row Level Security（RLS）により、データベースレベルでアクセス制御を実施
- 他組織のデータには一切アクセスできないセキュリティ設計
- 将来的なチーム利用・複数ユーザー招待にも対応可能なアーキテクチャ

### 5. セキュアなファイル管理

- アップロードされたファイルはSupabase Storageで安全に保管
- ファイルへのアクセスは組織単位のRLSポリシーで保護
- 組織ID/取引先IDによるフォルダ構造で、ファイルを体系的に整理
- ファイルサイズ制限（10MB）・ファイル形式バリデーションを実装済み
- アップロードトークンによる不正アクセス防止

### 6. 取引先管理機能

- 取引先の登録・削除をダッシュボードからワンストップで管理
- 取引先ごとのアップロードURLをワンクリックでコピー
- 取引先名で請求書を自動紐付け、どこからの請求かを即座に把握

### 7. 最新技術スタックによる高いパフォーマンスと拡張性

| レイヤー | 技術 | メリット |
|---|---|---|
| フロントエンド | Next.js 16 + React 19 | Server Components による高速レンダリング |
| スタイリング | Tailwind CSS 4 | 一貫性のあるモダンなデザインシステム |
| バックエンド | Supabase (PostgreSQL) | スケーラブルなBaaS、リアルタイム対応可能 |
| 認証 | Supabase Auth | メール/パスワード認証を標準搭載 |
| ストレージ | Supabase Storage | S3互換のスケーラブルなファイルストレージ |
| セキュリティ | Row Level Security | DBレベルでの堅牢なアクセス制御 |

### 8. 導入の手軽さ

- Webブラウザのみで利用可能（インストール不要）
- 無料でアカウント作成・即日利用開始
- 組織名・メールアドレス・パスワードの入力だけで数分でセットアップ完了

## 競合優位性まとめ

| 比較軸 | 請求受取太郎 | 従来型サービス |
|---|---|---|
| 取引先のアカウント | **不要** | 必要なことが多い |
| 導入までの時間 | **数分** | 数日〜数週間 |
| 取引先への展開 | **URLを送るだけ** | 招待・登録フローが必要 |
| 支払いステータス管理 | **ワンクリック** | 複数ステップが必要 |
| マルチテナント | **標準対応** | オプションや上位プラン |
| セキュリティ | **RLSによるDBレベル保護** | アプリレベルの制御が多い |

---

## 想定ターゲット

- **中小企業の経理担当者** ― 少人数で多数の取引先からの請求書を処理
- **フリーランス / 個人事業主** ― シンプルに請求書を受け取り・管理したい
- **スタートアップ** ― 軽量・高速に経理業務を立ち上げたい
- **BPO / 経理代行事業者** ― マルチテナントで複数クライアントを管理

## 今後の拡張可能性

- OCR / AI による請求書の自動読み取り（金額・日付の自動入力）
- 会計ソフト連携（freee, マネーフォワード等）
- 請求書の承認ワークフロー
- Slack / メール通知（新規請求書受信時）
- チームメンバーの招待・権限管理
- API提供による外部システム連携
